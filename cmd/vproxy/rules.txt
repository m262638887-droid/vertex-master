📡 关于匿名遥测（默认开启，你可以随时关闭）

  本软件会向作者发送匿名心跳，用于了解版本分布和活跃情况。
  这是纯被动的统计行为——服务端只接收数据，不会向你的程序下发任何指令。

  会发送的内容（全部为非敏感的运行环境信息）：
    • 匿名实例 ID（本地随机生成的 32 位字符，与你的身份无关）
    • 软件版本号 / 平台（如 linux/amd64）/ Go 运行时版本
    • 二进制文件 SHA256（用于识别正版与篡改版）
    • CPU 核心数 / 内存大小（GB）
    • 是否运行在容器内 / 是否在 Termux（Android）
    • 系统时区 / 系统语言（用于粗粒度的地理与语言分布统计）
    • 软件首次启动时间 / 启动次数

  不会发送的内容：
    ✗ IP 地址 / MAC 地址 / 主机名 / 用户名
    ✗ API 密钥 / 对话内容 / 请求内容
    ✗ 任何文件路径 / 进程列表 / 网络配置

  公开统计页面：https://stat.baimeow.icu

  如何关闭：
    编辑 config/config.json，添加或修改：
    "telemetry_enabled": false
    然后重启程序。
    或在管理面板「设置」页一键关闭。

🐳 Docker 部署同意方式

  Docker 容器内 stdin 通常无 TTY，无法交互式输入 yes。
  此种情况下程序会要求你提供同意文件：

    1. 在挂载到容器 /app/config 的本机目录下，创建文件：
         config/state/agreed-rules-docker.txt

    2. 文件内容写入程序首次启动时打印的"规则版本哈希"。
       一行命令搞定（哈希以程序输出为准）：
         echo <规则版本哈希> > ./config/state/agreed-rules-docker.txt

    3. 重启容器。

  注意：
    • 此机制仅在 Docker 容器内严格生效；裸机部署仍走交互式同意。
    • 规则更新后哈希会变化，需要按上述方式重新同意一次。
    • 同意文件不要提交到任何公开仓库——它是你"接受规则"的本地凭证。
