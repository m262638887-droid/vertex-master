## 📖 核心概念与准备工作

解压后，你会看到如下文件结构：

```
vertex-proxy（或 vertex-proxy.exe）   ← 主程序（仅此一个）
config/
  ├── config.example.json              ← 配置文件模板
  ├── api_keys.example.txt             ← API 密钥模板
  └── models.json                      ← 可用模型列表（一般无需修改）
```

还包含一个启动脚本（Windows 为 `启动.bat`，Linux/Android 为 `start.sh`），用于自动初始化环境。

> 🌟 **什么是 Web 管理面板？**
> 我们提供了一个直观的可视化后台！你可以轻松管理 API 密钥、一键导入代理配置（支持 vmess、vless、ss 等节点链接和订阅批量测速）、自定义模型别名，并修改系统设置（如重试次数、API 端口、登录密码等）。

### 快速上手只需两步

> ⚠️ **首次启动时**，程序会显示使用规则。请仔细阅读，输入 `yes` 同意后方可使用。
> **本软件完全免费**——如果你为它付了钱，你被骗了。正版获取：https://discord.gg/odysseia

**第一步：获取并牢记管理员密码**

首次启动时，程序会自动生成一个管理员密码，并**在终端/命令行窗口打印出来**，类似这样：

```
============================================================
[*] 首次启动，已自动生成管理员密码：
    密码: x3Kp9_aQ2mZv
    访问: http://<host>:<port>/admin
============================================================
```

妥善保管此密码。使用该密码登录管理面板（浏览器访问 `http://127.0.0.1:2156/admin/` 即可。

> **密码忘了怎么办？**
> 别担心，打开 `config/config.json` 文件，找到 `admin_password` 字段即可查看或修改。

**第二步：创建你的 API 密钥**

登录管理面板后，点击左侧的「密钥」菜单。在右侧填入名称（如 test）和自定义的 API 密钥（如 `sk-mykey123`），点击“添加”保存。
*(支持自定义，但必须以 `sk-` 开头，也可点击“✨”按钮随机生成)*

在你的 AI 客户端（如 Cherry Studio、SillyTavern 酒馆、ChatBox 等）中，进行如下配置：
- **API Key**: 填入你刚设置的密钥（例如 `sk-mykey123`）
- **Base URL / API 地址**: 填入 `http://127.0.0.1:2156/v1`

完成配置，即可开始使用！

## 💻 在电脑上运行（Windows）

1. **解压文件**：将下载的压缩包解压到任意目录，例如 `D:\vertex-proxy\`。

2. **启动程序**：双击运行 **`启动.bat`**。
   - 程序会自动初始化配置文件。
   - 会弹出一个控制台窗口，显示运行日志。**请勿关闭此窗口**，首次启动的管理员密码将在此显示。

3. **配置密钥**：登录管理面板（浏览器打开 `http://127.0.0.1:2156/admin/`），进入「密钥」页面，添加并保存你的 API 密钥。

4. **客户端连接**：在你的客户端配置相应的 API 地址和密钥即可使用。

**✨ 进阶：开机自启**
- **简单方法**：按 `Win + R`，输入 `shell:startup` 回车，打开「启动文件夹」，将 `启动.bat` 的快捷方式放入其中。
- **专业方法**：使用 [NSSM](https://nssm.cc/) 将其注册为系统服务。

**✨ 进阶：局域网访问**
若需让其他设备连接，请在 Windows 防火墙中放行 `2156` 端口。其他设备通过你电脑的局域网 IP（例如 `192.168.1.100`，而非 `127.0.0.1`）进行连接。

## 🐧 在服务器上运行（Linux）

推荐在 VPS 或云服务器上部署。

```bash
# 1. 解压文件
unzip vertex-proxy-linux-amd64.zip -d vertex-proxy
cd vertex-proxy

# 2. 赋予执行权限并启动（会自动初始化配置）
sh start.sh
```

首次启动时，请注意终端输出的管理员密码。随后在浏览器访问 `http://服务器IP:2156/admin/` 登录控制台，进入左侧「密钥」菜单添加 API Key。

### 推荐：使用 systemd 守护进程常驻后台

压缩包内附带了 `vertex-proxy.service` systemd 服务模板文件：

```bash
# 假设程序解压到了 /opt/vertex-proxy 目录
sudo cp vertex-proxy.service /etc/systemd/system/
sudo systemctl daemon-reload

# 设置开机自启并立即启动服务
sudo systemctl enable --now vertex-proxy

# 查看服务运行状态
sudo systemctl status vertex-proxy

# 查看实时日志（首次启动需在此查找管理员密码）
journalctl -u vertex-proxy -f
```

配置完成后，程序将开机自启，并在崩溃时自动重启，保障服务稳定运行。
> **注意**：如果你的安装路径不是 `/opt/vertex-proxy`，请务必修改 `vertex-proxy.service` 文件中的 `WorkingDirectory` 和 `ExecStart` 路径。

### 替代方案：简单后台运行

如果不使用 systemd，也可以使用 `nohup` 简单将其放入后台：

```bash
nohup ./vertex-proxy > vproxy.log 2>&1 &

# 查看日志（查找密码）
tail -f vproxy.log
```

### 可选：配置 Nginx 反向代理 (HTTPS 支持)

如果需要绑定域名并启用 HTTPS，可配置 Nginx 反向代理：

```nginx
location / {
    proxy_pass http://127.0.0.1:2156;
    proxy_http_version 1.1;
    proxy_set_header Connection "";
    proxy_buffering off;        # 【重要】流式输出必须关闭缓冲
    proxy_read_timeout 3600s;   # 设置长超时时间，防止长对话或大文件处理时连接断开
}
```

## 🍎 在 Mac 上运行（macOS）

Mac 上的部署流程跟 Linux 几乎一样，区别只在于：①下载哪个包；②首次运行需要绕过 Gatekeeper（系统安全检查）。

### 第一步：选对的包

Mac 分两种 CPU，下载前先确认：

- **Apple Silicon**（M1 / M2 / M3 / M4 等芯片，2020 年后的 Mac）→ 下载 `vertex-proxy-darwin-arm64.zip`
- **Intel 芯片**（2020 年前的 Mac）→ 下载 `vertex-proxy-darwin-amd64.zip`

不确定的话，点屏幕左上角苹果图标 → 「关于本机」，看「芯片」那一行就知道了。

### 第二步：解压并允许运行

打开「终端」（在「应用程序 → 实用工具 → 终端」里），定位到解压出来的目录：

```bash
cd ~/Downloads/vertex-proxy
chmod +x vertex-proxy

# 这一步很关键：清掉 Gatekeeper 的隔离标记，否则双击会被系统拦下
xattr -d com.apple.quarantine vertex-proxy 2>/dev/null || true
```

> 如果不执行 `xattr` 这一步，运行时会弹「无法验证开发者」的提示。也可以在「系统设置 → 隐私与安全性」里手动点「仍要打开」。

### 第三步：跑

```bash
./start.sh
```

第一次会打印一个随机的管理员密码，记下来——后面登录 Web 面板要用。

### 让它常驻后台

最简单的方式：

```bash
nohup ./vertex-proxy > vertex.log 2>&1 &
```

> 想开机自启可以写一个 launchd plist，但对大多数人来说 `nohup` 已经足够。

## 📱 在手机上运行（Android）

你可以在安卓手机上完美运行此程序。首先需要安装终端模拟器 **Termux**。

### 第一步：安装 Termux

强烈建议从 [F-Droid](https://f-droid.org/packages/com.termux/) 下载安装 Termux。**切勿使用 Google Play 商店版本**（该版本已停止维护，存在兼容性问题）。

### 第二步：配置环境并解压

打开 Termux，依次执行以下命令：

```bash
# 1. 更新软件包列表（可能需要等待，遇到提示直接回车默认即可）
pkg update -y
pkg install -y unzip

# 2. 授予存储访问权限（手机会弹出权限请求，请允许）
termux-setup-storage
```

将下载的压缩包移动到手机的下载目录，然后在 Termux 中继续执行：

```bash
# 3. 复制压缩包并解压（请将文件名替换为实际下载的文件名）
cp ~/storage/downloads/vertex-proxy-android-arm64.zip ~/
cd ~
unzip vertex-proxy-android-arm64.zip -d vertex-proxy
cd vertex-proxy

# 4. 赋予执行权限
chmod +x vertex-proxy
```

### 第三步：启动服务

```bash
sh start.sh
```

> **注意**：在 Termux 中使用 `sh start.sh` 而非 `./start.sh` 以避免权限问题。

首次启动时，记录屏幕上的管理员密码。然后在手机浏览器访问 `http://127.0.0.1:2156/admin/` 登录管理面板并添加 API 密钥。

### 解决安卓杀后台问题

为了防止系统自动清理后台进程，请执行以下操作：

1. 在 Termux 中运行唤醒锁命令：
   ```bash
   termux-wake-lock
   ```
2. 进入手机系统的「设置 → 电池/应用管理」，找到 Termux，将其设置为「不限制后台运行」或「忽略电池优化」（不同品牌手机设置路径可能略有不同）。

### 局域网内其他设备如何连接？

在 Termux 中输入 `ifconfig` 查看手机局域网 IP（通常为 `192.168.x.x`）。其他设备将 API 地址配置为 `http://192.168.x.x:2156/v1` 即可连接。

> **提示**：如果是普通的 64 位 Linux ARM 设备（例如装了 64 位系统的树莓派 4/5 等），请下载 `vertex-proxy-linux-arm64.zip`，部署方法与 Linux 服务器一致。

## 🐉 在鸿蒙上运行（HarmonyOS）

鸿蒙现在分两条线，**能不能跑差别很大**，先看你的手机是哪种：

### 哪些版本能跑

| 系统版本 | 能不能跑 | 怎么跑 |
|---|---|---|
| **HarmonyOS 4.x 及更早**（鸿蒙 4 / 鸿蒙 3 / 鸿蒙 2） | ✅ 能跑 | 用 Termux，参考[在手机上运行（Android）](#在手机上运行android) |
| **HarmonyOS NEXT**（纯血鸿蒙，2024 年起的鸿蒙 5） | ❌ 跑不了 | 见下方说明 |

判断方法：手机「设置 → 关于本机」看版本号。带 "HarmonyOS NEXT" 字样、或者发现**装不了任何 APK 文件**，那就是 NEXT。

### 老鸿蒙（4.x 及更早）怎么跑

老鸿蒙底层兼容 Android 应用框架，**Termux 能装、能跑 Android 架构二进制**。直接照[在手机上运行（Android）](#在手机上运行android)那一节做就行——下载 `vertex-proxy-android-arm64.zip`，操作完全一样。

> 小提示：部分华为机型对后台限制更严，做完 `termux-wake-lock` 后，记得在「电池 → 启动管理」里把 Termux 也手动允许"允许自启动 / 允许后台活动"。

### 纯血鸿蒙（NEXT）为什么跑不了

简单一句话：**它彻底不是 Linux 了，也不再兼容 Android 应用。**

技术上的原因：

1. **内核换了**：NEXT 用的是华为自研的 HarmonyOS 内核，不再是 Android 那套基于 Linux 的内核。我们打包出来的二进制（包括 `vertex-proxy-linux-arm64` 与 `vertex-proxy-android-arm64`）是 ELF 格式，依赖 Linux/Android 系统调用——NEXT 上没有这套接口，跑不动。
2. **应用层也换了**：NEXT 只能跑 ArkTS 写的 HAP 应用包（`.hap` 文件），不能装 APK，也不能装 Termux（Termux 本身是 Android 应用）。所以连"曲线救国塞进 Termux"这条路也走不通。
3. **想支持必须重写**：要在 NEXT 上跑，得用 ArkTS / C++ 做一个原生 HAP 应用，工作量约等于重做一个端，目前没这个计划。

如果你正好用的是 NEXT 手机，可行的临时方案有两个：

- **找一台老设备代跑**：在家里的旧安卓手机、平板、电脑、路由器上跑这个程序，鸿蒙手机连同一个 WiFi，API 地址填那台设备的局域网 IP（参考 [Android 章节最后一段](#局域网内其他设备如何连接)）。
- **跑在云服务器上**：买个最便宜的 VPS，按 [Linux 章节](#在服务器上运行linux) 跑起来，鸿蒙手机直接连云端的地址就行。

## ⚙️ 配置修改指南

绝大部分配置都可以在 Web 管理面板（`http://127.0.0.1:2156/admin/`）的「设置」页面进行可视化修改，修改后实时生效，无需重启。

如果需要手动编辑配置文件，文件路径为 `config/config.json`。配置文件修改后，程序每分钟会自动检测并加载新配置，同样无需重启。

基础配置项如下：

```json
{
  "port_api": 2156,
  "max_retries": 2,
  "admin_password": "",
  "proxy_url": ""
}
```

### 核心配置项说明

| 参数名 | 默认值 | 作用说明 |
|--------|--------|----------|
| `port_api` | `2156` | 服务监听端口。 |
| `admin_password` | 自动生成 | Web 管理面板登录密码。留空则下次启动时自动生成新密码。 |
| `max_retries` | `10` | 请求失败时的自动重试次数。网络不稳定时可适当调大。 |
| `proxy_url` | 空 | 出站代理服务器地址（详情见下节）。 |
| `force_no_stream`| `false` | 设置为 `true` 时，强制将所有流式请求转为非流式返回。 |

### 模型别名与伪装流式

`config/models.json` 文件定义了系统支持的模型列表，通常无需修改。

**配置模型别名**：
如果你的客户端仅支持调用 `gpt-4o` 等特定名称，可以在管理面板的「模型」页面配置别名映射：
`gpt-4o`  →  `gemini-3.5-flash`
配置后，客户端调用 `gpt-4o` 时，系统会自动代理到 `gemini-3.5-flash`。

**假流式 (Fake Stream)**：
部分客户端强制要求流式输出。你可以在模型名称前加上 `假流式-` 或 `fake-` 前缀（例如 `fake-gemini-2.5-pro`），程序将在获取完整回复后，模拟打字机效果逐段推送给客户端。

## 🎛 Web 管理面板使用教程

程序自带的管理面板是你配置和维护 Vertex AI Proxy 的核心工具。登录后，你会在左侧看到以下菜单：

### 1. 📊 概览页面
- **作用**：查看服务的当前运行状态。
- **展示信息**：包括并发池的状态、当前已缓存的 reCAPTCHA token 数量、拦截规则等。你可以在此随时点击“刷新”查看最新状态。

### 2. ⚙️ 设置页面
- **作用**：可视化修改核心配置，**修改后立即生效，无需重启程序**。
- **常见操作**：
  - **修改监听端口**：如果你想修改默认的 `2156` 端口。
  - **修改管理员密码**：将复杂的随机密码修改为你容易记忆的密码。
  - **调整重试次数 (max_retries)**：网络环境较差时可调高至 `10`。
  - **强制非流式**：勾选后所有请求将被强制转换为非流式（某些客户端强制要求）。
  - **配置全局代理**：直接填写 Clash/v2ray 的代理地址（如 `http://127.0.0.1:7890`）。

### 3. 🔑 密钥页面 (必须配置)
- **作用**：管理客户端连接所需的 API Key。
- **使用方法**：
  1. 在输入框填入名称（随意填写，如 `手机使用`）和密钥（如 `sk-test123`，必须以 `sk-` 开头）。
  2. 也可以点击输入框右侧的“✨”按钮，让系统为你随机生成一个安全的密钥。
  3. 填好后点击“添加”，密钥即刻生效。

### 4. 🧩 模型页面
- **作用**：配置模型清单与别名映射。
- **适用场景**：很多第三方客户端（如 OpenAI 的客户端）默认只支持调用 `gpt-4o` 或 `gpt-3.5-turbo`。
- **如何映射**：
  在“别名映射”文本框中输入，每行一个：
  ```
  gpt-4o=gemini-3.5-flash
  gpt-3.5-turbo=gemini-2.5-pro
  ```
  这样，当客户端请求 `gpt-4o` 时，代理会自动将其转发给后端的 `gemini-3.5-flash` 模型。

### 5. 🌐 订阅与并发页面 (解决网络问题)
- **作用**：配置节点，解决国内无法直连 Google API 的网络限制问题。内置 mihomo 内核，**不用安装额外代理软件**。同时能极大缓解单 IP 的 429 请求频率限制，并在“PVP（高频竞争）”场景下显著提升响应优势。
- **如何导入**：
  - **导入文件**：点击上传按钮导入导出的节点配置文件（.yaml, .txt 等）。
  - **导入订阅**：在“订阅链接”框中填入你的机场订阅链接，点击“拉取并保存”。
- **并发竞速**：导入节点后，系统会自动测试节点连通性。如果开启了“并发池”，程序发请求时会同时通过多个健康的节点发出，**谁先返回就用谁的结果**。这可以极大提高返回速度和成功率。

### 6. 🎨 外观页面
- **作用**：个性化你的控制台背景。支持输入图片 URL 或从本地上传图片，设置仅对当前浏览器生效。

## 🌐 网络问题与节点代理

为了正常访问 Google API，你需要确保网络畅通。如果直连失败（例如在国内环境），提供以下三种解决方案：

### 方法一：配置全局代理地址

如果本机已运行代理软件（如 Clash、V2Ray 等），可在管理面板「设置」页或 `config.json` 中配置代理地址：

```json
{
  "proxy_url": "socks5://127.0.0.1:1080" 
}
```
或使用 HTTP 代理（如 `http://127.0.0.1:7890`），具体端口取决于你的代理软件配置。

### 方法二：一键导入订阅节点（强烈推荐）

在管理面板的「代理与节点」页面，直接粘贴你的代理订阅链接或节点链接（支持 vmess、vless、shadowsocks、trojan 协议）。
**程序内置了 mihomo 内核，会自动解析配置并建立连接，无需额外安装或运行其他代理软件。**

### 方法三：启用并发池 (竞速模式)

如果你导入了多个可用节点，可以在管理面板开启「并发池」功能。开启后，程序将同时使用多个节点发起请求，并采用最先返回的有效结果。此功能可显著提升响应速度和可用性。

> 💡 **提示**：在手机或服务器等无桌面环境下部署时，使用方法二和方法三最为便捷。

## ❓ 常见问题解答（FAQ）

**Q: 所有的请求都提示 401 错误或 "missing API key" 怎么办？**
A: 请确认：
1. 已在管理面板中成功添加了 API 密钥。
2. 客户端软件中填写的 API Key 与管理面板中配置的完全一致。

**Q: 忘记管理员密码，无法登录管理面板怎么办？**
A: 使用文本编辑器打开 `config/config.json` 文件，查看 `admin_password` 字段的值即可。登录后，可在面板「设置」页修改为方便记忆的密码。

**Q: 请求一直报错、频繁触发 429 错误或认证失败怎么办？**
A: 这通常意味着你的出口 IP 被 Google 限制或拉黑。
1. 请参考上述「网络问题与节点代理」章节，配置或更换代理节点。
2. 可尝试在设置中将 `max_retries`（重试次数）适当调大（例如调整为 `10`）。

**Q: 为什么同一局域网下的其他设备无法连接？**
A: 请检查以下两项：
1. 运行程序的电脑/服务器防火墙是否已放行 `2156` 端口。
2. 其他设备配置的 API 地址必须使用运行程序的设备的**局域网 IP**（例如 `192.168.1.x`），绝对不能填 `127.0.0.1`（此地址仅限本机访问自身）。

**Q: 如何修改默认的 2156 监听端口？**
A: 修改 `config/config.json` 文件中的 `port_api` 字段值为你想要的端口，保存后重启程序即可生效。

**Q: 只有 `/v1/chat/completions` 能用吗？**
A: 不仅如此，程序全面兼容主流的多模态体验，支持以下 OpenAI 格式的端点：
- 聊天：`/v1/chat/completions`
- 模型查看：`/v1/models`
- 图像生成与编辑：`/v1/images/generations`、`/v1/images/edits`、`/v1/images/variations`
- 语音合成 (TTS)：`/v1/audio/speech`
同时，也支持 `v1beta/models` 和原生 Gemini 接口的透传形式。
